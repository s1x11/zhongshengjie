6. 评估与众生界项目的融合程度

### 1.2 场景类型扩展（24→28种）

**原始24种场景**：开篇、战斗、情感、悬念、转折、世界观展开、...(其他18种)

| 向量化 | `--sync` | Qdrant: 387,377 条向量 |

### 2.2 配置统一重构

**重构目标**: 将 `case_builder.py` 的硬编码配置改为使用 `config_loader` 统一管理。

**修改内容**:

```python
# 新增导入
from config_loader import (
    get_config,
    get_qdrant_url,
    get_model_path,
    get_case_library_dir,
    get_collection_name,
    get_novel_sources,
)

# CaseBuilder.__init__ 重构
if HAS_CONFIG_LOADER:
    self.qdrant_url = get_qdrant_url()
    self.collection_name = get_collection_name("case_library")
    self.model_path = get_model_path()
    self.novel_sources = get_novel_sources()
```

**重构前问题**:
- 硬编码 Qdrant URL: `http://localhost:6333`
- 硬编码 Collection 名: `case_library_v2`
- 硬编码模型路径: 环境变量 `BGE_M3_MODEL_PATH`
- 硬编码小说路径: 命令行参数

**重构后优势**:
- 统一通过 `config.json` 管理
- 自动读取 `novel_sources.directories` 配置
- 与其他组件配置一致

### 2.3 融合度分析

**原始融合度**: 42/100（严重断裂）

**诊断问题**:
- `case_builder.py` 未使用 `config_loader`
- 配置来源混乱（命令行 + 环境变量 + 硬编码）
- 与统一API层脱节

**重构后融合度**: 约 60/100

**提升原因**:
- 配置统一完成
- 自动读取 `config.json` 中的 `novel_sources`
- 与其他工具配置方式一致

**剩余差距**:
- `case_builder.py` 独立性强，不需要统一API层
- 不需要世界观适配（案例库是通用资源）
- 不需要场景契约（构建阶段，非创作阶段）
