# 众生界系统演进路线图

**版本**: v1.0  
**日期**: 2026-04-13  

---

## 总体愿景

从单机创作工具 → 多项目创作平台 → 可选云端协作平台

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│   Phase 1    │    │   Phase 2    │    │   Phase 3    │    │   Phase 4    │
│  架构夯实期  │ →  │  功能完善期  │ →  │  扩展准备期  │ →  │  平台化演进  │
│  1-2个月     │    │  2-3个月     │    │  3-6个月     │    │  6-12个月    │
└──────────────┘    └──────────────┘    └──────────────┘    └──────────────┘
```

---

## Phase 1: 架构夯实期 (1-2个月)

**目标**: 消除技术债务，建立清晰的模块边界

### 任务清单

| 任务 | 优先级 | 预计工时 | 验收标准 |
|------|--------|----------|----------|
| 消除模块间循环依赖 | P0 | 3天 | import-linter检查通过 |
| 完善单元测试覆盖 | P0 | 5天 | 覆盖率≥85% |
| 统一配置管理 | P1 | 2天 | 单一配置入口，废弃旧接口 |
| 日志规范化 | P1 | 2天 | 统一日志格式和级别 |
| 文档完善 | P2 | 3天 | 每个模块有README |

### 关键里程碑

- [ ] M1.1: import-linter规则配置完成并CI集成
- [ ] M1.2: 测试覆盖率从83.7%提升至85%+
- [ ] M1.3: 配置加载统一为单一入口
- [ ] M1.4: 模块边界文档完成

### 风险

| 风险 | 缓解 |
|------|------|
| 重构可能引入新bug | 先加测试再重构 |
| 循环依赖可能比预期复杂 | 使用工具辅助分析 |

## Phase 2: 功能完善期 (2-3个月)

**目标**: 完善核心功能，建立数据回流闭环

### 任务清单

| 任务 | 优先级 | 预计工时 | 验收标准 |
|------|--------|----------|----------|
| 实现评估反馈→技法库回流 | P0 | 5天 | 评估问题自动沉淀 |
| 实现用户修改→经验沉淀 | P1 | 4天 | diff分析可用 |
| 实现技法效果→推荐优化 | P1 | 3天 | 推荐权重自动调整 |
| 工作流状态持久化 | P1 | 3天 | 支持中断恢复 |
| 检索性能优化 | P2 | 3天 | P95延迟<100ms |
| 错误恢复机制完善 | P2 | 2天 | 常见错误可自动恢复 |

### 关键里程碑

- [ ] M2.1: 数据回流三级闭环全部打通
- [ ] M2.2: 工作流支持断点续作
- [ ] M2.3: 检索延迟满足性能目标
- [ ] M2.4: 错误恢复覆盖率≥80%

### 数据回流架构

```
创作输出 ──评估──→ 问题模式 ──提取──→ 新技法约束
     │                                      │
     └──用户修改──→ diff分析 ──沉淀──→ 经验日志  │
                                              │
技法推荐 ──使用──→ 创作结果 ──评分──→ 效果追踪 ──┘
```

## Phase 4: 平台化演进 (6-12个月)

**目标**: 可选的多用户协作和云端部署

### 任务清单

| 任务 | 优先级 | 预计工时 | 验收标准 |
|------|--------|----------|----------|
| 多用户隔离 | P0 | 10天 | 用户间数据完全隔离 |
| API服务化 | P1 | 8天 | RESTful API可用 |
| 云端部署方案 | P1 | 5天 | Docker一键部署 |
| 分布式向量检索 | P2 | 8天 | Qdrant集群方案 |
| 权限与协作 | P2 | 5天 | 基础RBAC |

### 关键里程碑

- [ ] M4.1: 多用户系统上线
- [ ] M4.2: API接口文档完成
- [ ] M4.3: Docker Compose部署可用
- [ ] M4.4: 向量检索可水平扩展

### 技术选型预研

| 组件 | 当前 | 候选升级 |
|------|------|----------|
| 应用框架 | CLI脚本 | FastAPI |
| 向量数据库 | Qdrant单节点 | Qdrant集群 |
| 缓存 | 无 | Redis |
| 消息队列 | 无 | Redis Streams |
| 容器化 | 无 | Docker Compose → K8s |
| 数据库 | SQLite/JSON文件 | PostgreSQL |
