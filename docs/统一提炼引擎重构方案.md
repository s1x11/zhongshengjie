### 核心问题
1. **数据流断裂**：提炼+拆书分离，配置分散，数据无法回流学习
2. **对话式缺失**：无状态持久化，无进度反馈，无错误恢复

## 一、自动维护功能盘点

### 1.1 提炼模式变更：素材提炼模式

> **核心理念变更**：技法管理 → 用户提供资料 → 系统提炼技法

- 用户不清楚什么是"技法"，只会提供资料（小说、片段、示例）

- 用户无需理解技法概念，只需提供优秀素材

**新模式流程**：

## 二、核心理念：单一入口

### 2.1 问题根源

**当前设计的问题**：
- 拆书（`case_builder.py`）和提炼（`.novel-extractor/`）是**两个独立入口**
- 用户需要分别运行两个工具
- 数据提取流程不一致
- 维度进度追踪分散

**本质**：拆书就是提炼，两者都是"从外部小说库提取数据入库"的过程。

### 2.2 统一设计原则

| 原则 | 说明 |
|------|------|
| **单一入口** | 一个函数调用启动所有提取 |
| **并行执行** | 11个维度同时提取，不顺序等待 |
| **统一进度** | 所有维度共用同一进度追踪 |
| **统一入库** | 所有维度同时写入Qdrant |

### 2.3 目标架构

```
外部小说库 (E:\小说资源)
        │
        ▼
┌───────────────────────────────────────────┐
│        统一提炼引擎                        │
│        tools/unified_extractor.py         │
│                                           │
│  ┌─────────────────────────────────────┐ │
```

### 5.2 数据提炼触发机制（修正版）

> **设计变更**：数据提炼不应作为工作流阶段，而应作为独立功能，通过对话触发或后台增量执行。

#### 5.2.1 触发方式

| 触发方式 | 说明 | 使用场景 |
|----------|------|----------|
| **对话主动触发** | 用户说"提炼数据"/"初始化数据" | 首次使用、大量新增小说 |
| **后台自动增量** | IncrementalSyncManager检测变化后自动执行 | 日常增量同步 |
| **数据新鲜度提示** | 检测到新小说时主动提示用户 | 确保数据及时更新 |

#### 5.2.2 对话触发流程

```
用户说："提炼数据" 或 "初始化数据"
        ↓
ConversationEntryLayer
├── IntentIdentifier: intent = "data_extraction"
├── 判断提取类型：
│   ├── "提炼数据"/"初始化数据" → 全面提炼
│   └── "增量提炼"/"同步新小说" → 增量提炼
└── 执行 UnifiedExtractor
    ├── 全面提炼：扫描全部6,237本小说
    └── 增量提炼：只处理新增小说
        ↓
返回提取结果：
├── 提取维度数
├── 新增数据量
├── 发现新场景类型（如有）
└── 耗时
```

#### 5.2.3 数据新鲜度检测

```python
class DataFreshnessChecker:
    """数据新鲜度检测器"""
    
    def check_freshness(self) -> dict:
        """
        检测数据新鲜度
        
        Returns:
            {
                "is_fresh": False,
                "new_novels_count": 3,
                "last_extraction": "2026-04-08 10:00:00",
                "suggestion": "检测到3本新小说，是否提炼？"
            }
        """
        new_novels = self._detect_new_novels()
        last_extraction = self._get_last_extraction_time()
        
        if new_novels:
            return {
                "is_fresh": False,
                "new_novels_count": len(new_novels),
                "last_extraction": last_extraction,
                "suggestion": f"检测到{len(new_novels)}本新小说，是否提炼？"
            }
        
        return {"is_fresh": True}
```


```python
class ChangeDetector:
    """统一变更检测器"""
    
    def __init__(self):
        self.watch_list = {
            "outline": "总大纲.md",
            "settings": "设定/*.md",
            "techniques": "创作技法/**/*.md",
            "tracking": "设定/hook_ledger.md",
        }
    
    def scan_changes(self) -> Dict[str, List[Change]]:
        """扫描所有数据源变更"""
        changes = {}
        for source, pattern in self.watch_list.items():
            changes[source] = self._detect_file_changes(pattern)
        return changes
    
    def sync_changes(self, changes: Dict[str, List[Change]]):
        """同步变更到对应的存储"""
        if changes.get("outline"):
            self._sync_outline_to_worldview()
        if changes.get("settings"):
            self._sync_settings_to_graph()
        if changes.get("techniques"):
            self._sync_techniques_to_qdrant()
```

### 9.8 解决方案：会话意图识别与自动更新

```python
class ConversationDataExtractor:
    """会话数据提取器"""
    
    # 意图类型映射
    INTENT_PATTERNS = {
        "add_character_ability": [
            r"(.+?)有个新能力叫(.+)",
            r"(.+?)的血脉能力是(.+)",
            r"(.+?)获得了(.+)能力",
        ],
        "add_faction": [
            r"加一个新势力叫(.+)",
            r"有个新势力叫(.+)",
            r"新增势力(.+)",
        ],
        "modify_plot": [
            r"剧情改成(.+)",
            r"(.+)的剧情是(.+)",
            r"(.+)改成(.+)",
        ],
        "add_power_level": [
            r"力量体系加个新境界(.+)",
            r"(.+)的力量境界是(.+)",
        ],
    }
    
    def extract_and_update(self, user_input: str) -> Dict[str, Any]:
        """从对话中提取信息并更新"""
        # 1. 意图识别
        intent, entities = self._identify_intent(user_input)
        
        if not intent:
            return {"status": "no_update_needed"}
        
        # 2. 提取结构化信息
        structured_data = self._extract_structured_data(intent, entities)
        
        # 3. 更新对应文件
        update_result = self._update_source_file(intent, structured_data)
        
        # 4. 同步到向量库
        sync_result = self._sync_to_vectorstore(intent, structured_data)
        
        return {
            "status": "updated",
            "intent": intent,
            "data": structured_data,
            "file_updated": update_result,
            "vectorstore_synced": sync_result,
        }
```

### 9.12 新增设定文档处理机制

**场景**：用户可能随时添加新的设定文档，系统需要自动识别并处理。

```python
class NewSettingHandler:
    """新增设定文档处理器"""
    
    def detect_new_files(self) -> List[Path]:
        """检测设定目录新增的MD文件"""
        existing = self._load_registered_files()
        current = set(Path("设定").glob("*.md"))
        return list(current - existing)
    
    def classify_new_file(self, file_path: Path) -> str:
        """自动分类新设定文件"""
        content = file_path.read_text(encoding="utf-8")
        
        # 分类规则
        if any(kw in content for kw in ["角色", "人物", "主角"]):
            return "character_setting"
        elif any(kw in content for kw in ["势力", "宗门", "派系"]):
            return "faction_setting"
        elif any(kw in content for kw in ["力量", "血脉", "境界"]):
            return "power_setting"
        elif any(kw in content for kw in ["技术基础", "体系详解"]):
            return "technical_reference"  # 不入库
        else:
            return "general_setting"
    
    def auto_register(self, file_path: Path, category: str):
        """自动注册新设定文件"""
        if category in ["character_setting", "faction_setting", "power_setting"]:
            # 入库处理
            self._sync_to_vectorstore(file_path)
        
        # 注册到配置
        self._register_file(file_path, category)
        
        # 生成意图类型（可选）
        self._generate_intent_patterns(file_path, category)
```

## 十、对话式工作流特殊需求（新增）

> **背景**：本项目主要通过对话方式使用工作流，而非运行Python脚本。需要针对对话式交互设计特殊机制。

### 10.1 对话式工作流的独特挑战

| 挑战 | 问题描述 | 影响 |
|------|----------|------|
| **状态追踪** | 用户中途离开，回来后系统不知进度 | 工作流中断无法恢复 |
| **多轮澄清** | 用户表达模糊，意图不明确 | 系统可能误解并执行错误操作 |
| **实时反馈** | 用户不知道当前在哪个阶段 | 用户焦虑，以为系统死机 |
| **错误恢复** | 系统理解错了，用户无法纠正 | 错误操作已写入文件，无法撤销 |
| **上下文延续** | 对话上下文有长度限制 | 长对话后系统"忘记"之前内容 |
| **主动引导** | 系统不会主动问用户缺少什么 | 信息缺失导致创作质量差 |

```python
class MissingInfoDetector:

        "start_chapter": {
            "chapter_outline": {
                "check": lambda chapter: Path(f"章节大纲/第{chapter}章大纲.md").exists(),
                "message": "章节大纲文件不存在",
                "suggestion": "先提供大纲，或让我根据总大纲生成"
            },
            "character_settings": {
                "check": lambda: Path("设定/人物谱.md").exists(),
                "message": "角色设定文件不存在",
                "suggestion": "请先创建角色设定"
            }
        }
    }
    
    def detect_missing(self, intent: str, entities: dict) -> list:
        """
        检测缺失信息
        
        Returns:
            [
                {
                    "type": "chapter_outline",
                    "message": "第一章大纲文件不存在",
                    "suggestion": "先提供大纲，或让我根据总大纲生成",
                    "severity": "warning"  # critical, warning, info
                }
            ]
        """
        
    def generate_missing_prompt(self, missing: list) -> str:
        """生成缺失信息提示"""
        lines = ["我注意到以下问题，可能影响创作质量："]
        for i, m in enumerate(missing, 1):
            icon = {"critical": "❌", "warning": "⚠️", "info": "ℹ️"}[m["severity"]]
            lines.append(f"{icon} {m['message']}")
        
        lines.append("\n建议：")
        suggestions = list(set(m["suggestion"] for m in missing))
        for i, s in enumerate(suggestions, 1):
            lines.append(f"{i}. {s}")
        lines.append("\n您希望如何处理？")
        
        return "\n".join(lines)
```

```python
class ProgressReporter:
    """进度报告器 - 实时反馈工作流进度"""
    
    PHASE_NAMES = {
        0: "需求澄清",
        1: "章节大纲解析",
        2: "场景类型识别",
        2.3: "数据提炼",
        2.5: "经验检索",
        3: "设定自动检索",
        3.5: "场景契约提取",
        4: "逐场景创作",
        5: "整章整合",
        6: "整章评估",
        7: "用户确认",
        8: "经验写入"
    }
    
    def generate_progress(self, state: dict) -> str:
        """生成进度报告"""
        current = state["current_phase"]
        total = state["total_phases"]
        phase_name = self.PHASE_NAMES.get(current, f"阶段{current}")
        
        # 进度条
        progress_bar = self._generate_progress_bar(current, total)
        
        report = f"""
【工作流进度】
{progress_bar}
当前：[{current}/{total}] {phase_name}
章节：第{state.get('chapter', '?')}章
状态：{'进行中' if state.get('in_progress') else '等待中'}
        """.strip()
        
        # 如果在场景创作阶段，显示更详细的信息
        if current == 4 and "scene_progress" in state:
            scene_prog = state["scene_progress"]
            report += f"\n场景：{scene_prog['current']}/{scene_prog['total']}"
        
        return report
    
    def _generate_progress_bar(self, current: int, total: int, 
                                width: int = 20) -> str:
        """生成进度条"""
        filled = int(width * current / total)
        bar = "█" * filled + "░" * (width - filled)
        percent = int(100 * current / total)
        return f"[{bar}] {percent}%"
```

```python
class EvaluationFeedbackLoop:
    """评估反馈回流器 - 让系统从评估中学习"""
    
    # ========== 触发阈值配置 ==========
    FEEDBACK_THRESHOLDS = {
        "technique_extraction": 8.5,   # 评分≥8.5才提取技法
        "case_extraction": 8.0,        # 评分≥8.0才提取案例
        "forbidden_detection_count": 3, # 同一问题出现3次才加入禁止项
        "similarity_threshold": 0.85,   # 相似度阈值（用于去重）
    }
    
    def __init__(self, qdrant_client, settings_dir: str):
        self.client = qdrant_client
        self.settings_dir = Path(settings_dir)
        self.feedback_history = []  # 反馈历史（用于去重和冲突检测）
        
    def process_evaluation(self, chapter: int, evaluation_result: dict):
        """
        处理评估结果，提取有价值的信息
        
        Args:
            evaluation_result: Evaluator返回的评估结果
                {
                    "scores": {...},
                    "issues": [...],
                    "highlights": [...],
                    "suggestions": [...]
                }
        """
        overall_score = evaluation_result.get("scores", {}).get("overall", 0)
        
        # 1. 提取有效写法 → 入库技法库（带阈值检查）
        if overall_score >= self.FEEDBACK_THRESHOLDS["technique_extraction"]:
            self._extract_good_practices(evaluation_result["highlights"])
        
        # 2. 提取禁止项（带计数阈值）
        self._extract_forbidden_items(evaluation_result["issues"])
        
        # 3. 提取好案例 → 入库案例库（带阈值检查）
        if overall_score >= self.FEEDBACK_THRESHOLDS["case_extraction"]:
            self._extract_good_cases(evaluation_result["highlights"])
        
        # 4. 生成作家改进建议
        self._generate_improvement_suggestions(evaluation_result)
        
    # ========== 去重机制 ==========
    def _deduplicate_technique(self, technique: dict) -> bool:
        """
        检查技法是否已存在（去重）
        
        Returns:
            True: 已存在，跳过
            False: 不存在，可以添加
        """
        # 1. 使用向量相似度检测
        similar = self.client.search(
            collection_name="writing_techniques_v2",
            query_vector=self._get_embedding(technique["example"]),
            limit=5
        )
        
        for result in similar:
            if result.score >= self.FEEDBACK_THRESHOLDS["similarity_threshold"]:
                return True  # 已存在相似技法
        
        return False
    
    # ========== 冲突解决机制 ==========
    def _resolve_feedback_conflict(self, new_feedback: dict, existing: dict) -> dict:
        """
        解决反馈冲突
        
        场景：多个评估对同一技法有不同意见
        策略：
        1. 优先保留评分更高的版本
        2. 合并不同维度的反馈
        3. 标记为"需人工确认"如果无法自动解决
        """
        if new_feedback.get("score", 0) > existing.get("score", 0):
            # 新反馈评分更高，替换
            return {**new_feedback, "replaced": existing}
        elif new_feedback.get("dimension") != existing.get("dimension"):
            # 不同维度的反馈，合并
            return {
                **existing,
                "dimensions": list(set(
                    existing.get("dimensions", [existing.get("dimension")]) + 
                    [new_feedback.get("dimension")]
                ))
            }
        else:
            # 无法自动解决，标记人工确认
            return {
                **existing,
                "conflict": True,
                "conflicting_feedback": new_feedback,
                "needs_manual_review": True
            }
    
    def _extract_good_practices(self, highlights: list):
        """提取有效写法（带去重）"""
        for h in highlights:
            if h.get("type") == "technique_usage":
                technique = {
                    "name": h["technique_name"],
                    "dimension": h["dimension"],
                    "example": h["content"],
                    "effect": h.get("effect", ""),
                    "source_chapter": h.get("chapter"),
                    "auto_extracted": True,
                    "score": h.get("score", 0)
                }
                
                # 去重检查
                if not self._deduplicate_technique(technique):
                    self._upsert_technique(technique)
                    
    def _extract_forbidden_items(self, issues: list):

        # 统计问题出现次数
        issue_counts = {}
        for issue in issues:

        # 只有达到阈值才添加
        for pattern, count in issue_counts.items():
```

```python
class ChapterExperienceLogger:
    """章节经验记录器 - 让系统记住创作经验"""
    
    def __init__(self, log_dir: str = "章节经验日志"):
        self.log_dir = Path(log_dir)
        self.log_dir.mkdir(exist_ok=True)
        
    def log_experience(self, chapter: int, scene_type: str, 
                       writer: str, experience: dict):
        """
        记录创作经验
        
        Args:
            experience: {
                "techniques_used": [...],      # 使用的技法
                "techniques_effect": {...},    # 技法效果
                "user_modifications": [...],   # 用户修改
                "lessons_learned": [...],      # 经验教训
                "consistency_issues": [...],   # 一致性问题
                "what_worked": [...],          # 成功的做法
                "what_didnt_work": [...]       # 失败的做法
            }
        """
        log_file = self.log_dir / f"第{chapter}章_{scene_type}_{writer}.json"
        
        log_entry = {
            "chapter": chapter,
            "scene_type": scene_type,
            "writer": writer,
            "timestamp": datetime.now().isoformat(),
            **experience
        }
        
        # 追加到日志文件
        if log_file.exists():
            logs = json.loads(log_file.read_text(encoding="utf-8"))
        else:
            logs = []
        logs.append(log_entry)
        log_file.write_text(json.dumps(logs, ensure_ascii=False, indent=2))
        
    def retrieve_experience(self, scene_type: str, 
                           before_chapter: int,
                           writer: str = None) -> list:
        """
        检索相似场景的经验
        
        用于指导当前创作，避免重复犯错
        """
        experiences = []
        
        for log_file in self.log_dir.glob("*.json"):
            # 解析文件名：第X章_场景类型_作家.json
            parts = log_file.stem.split("_")
            if len(parts) >= 3:
                chapter = int(parts[0][1:])  # 提取章节号
                file_scene = parts[1]
                file_writer = parts[2]
                
                # 筛选条件
                if chapter >= before_chapter:
                    continue
                if scene_type and file_scene != scene_type:
                    continue
                if writer and file_writer != writer:
                    continue
                    
                # 读取经验
                logs = json.loads(log_file.read_text(encoding="utf-8"))
                experiences.extend(logs)
        
        # 按相关性排序（可使用向量检索优化）
        return experiences[:10]  # 返回最相关的10条
        
    def aggregate_lessons(self, scene_type: str = None) -> dict:
        """
        汇总经验教训
        
        用于生成作家改进报告
        """
        all_experiences = self.retrieve_experience(
            scene_type=scene_type, 
            before_chapter=999
        )
        
        aggregated = {
            "common_issues": Counter(),
            "effective_techniques": Counter(),
            "failed_approaches": Counter(),
            "user_modifications": []
        }
        
        for exp in all_experiences:
            for issue in exp.get("consistency_issues", []):
                aggregated["common_issues"][issue] += 1
            for tech in exp.get("techniques_used", []):
                if exp.get("techniques_effect", {}).get(tech) == "success":
                    aggregated["effective_techniques"][tech] += 1
            for approach in exp.get("what_didnt_work", []):
                aggregated["failed_approaches"][approach] += 1
                
        return aggregated
```

### 14.2 遗漏L1：场景-作家映射自动生成

#### 14.2.1 问题描述

**当前状态**：
```
发现新场景类型 "交易场景"
    ↓
更新 scene_types.json ✅
    ↓
但是 scene_writer_mapping.json 未更新 ❌
    ↓
工作流阶段2识别到"交易场景"
    ↓
查找 scene_writer_mapping.json → 找不到对应作家 ❌
    ↓
调度失败
```

**影响**：
- 新发现的场景类型无法被正确调度
- 工作流阶段2（场景类型识别）后无法分配作家
- 用户需要手动修改 `scene_writer_mapping.json`

#### 14.2.2 解决方案：SceneWriterMappingGenerator

```python
class SceneWriterMappingGenerator:
    """场景-作家映射自动生成器"""
    
    # 作家专长规则库
    WRITER_SPECIALTY_RULES = {
        "战斗": {"writer": "剑尘", "keywords": ["战斗", "对决", "厮杀", "交锋", "厮杀"]},
        "情感": {"writer": "墨言", "keywords": ["情感", "爱情", "亲情", "友情", "告白"]},
        "剧情": {"writer": "玄一", "keywords": ["转折", "伏笔", "悬念", "揭秘", "反转"]},
        "世界观": {"writer": "苍澜", "keywords": ["势力", "世界观", "设定", "体系", "规则"]},
        "意境": {"writer": "云溪", "keywords": ["意境", "诗意", "氛围", "描写", "景色"]},
    }
    
    # 场景关键词映射
    SCENE_KEYWORD_PATTERNS = {
        "战斗": ["战斗", "对决", "厮杀", "交锋", "战斗场景"],
        "情感": ["情感", "告白", "恋爱", "分手", "重逢"],
        "开篇": ["开篇", "序章", "第一章", "开始"],
        "转折": ["转折", "反转", "揭秘", "真相"],
        "交易": ["交易", "买卖", "交换", "谈判"],
        "日常": ["日常", "生活", "修炼", "训练"],
    }
    
    def generate_mapping(self, scene_type: str, 
                        discovered_cases: List[dict] = None) -> dict:
        """
        自动生成场景-作家映射
        
        Args:
            scene_type: 新发现的场景类型名称
            discovered_cases: 从外部小说库发现的该场景类型案例
            
        Returns:
            {
                "scene_type": "交易场景",
                "writer": "玄一",  # 根据关键词自动判断
                "phase": 1,
                "poetry_required": False,
                "description": "交易、谈判、买卖相关场景",
                "keywords": ["交易", "买卖", "谈判"],
                "confidence": 0.85
            }
        """
        # 1. 提取场景关键词
        keywords = self._extract_keywords(scene_type, discovered_cases)
        
        # 2. 匹配作家专长
        writer_match = self._match_writer(keywords)
        
        # 3. 判断是否需要诗词
        poetry_required = self._check_poetry_requirement(scene_type, keywords)
        
        # 4. 生成描述
        description = self._generate_description(scene_type, keywords, discovered_cases)
        
        return {
            "scene_type": scene_type,
            "writer": writer_match["writer"],
            "phase": 1,  # 默认阶段1
            "poetry_required": poetry_required,
            "description": description,
            "keywords": keywords,
            "confidence": writer_match["confidence"],
            "auto_generated": True
        }
    
    def _match_writer(self, keywords: List[str]) -> dict:
        """根据关键词匹配最合适的作家"""
        scores = {}
        
        for writer, rule in self.WRITER_SPECIALTY_RULES.items():
            score = sum(1 for kw in keywords if kw in rule["keywords"])
            scores[writer] = score
        
        if not scores or max(scores.values()) == 0:
            # 无匹配，默认使用玄一（剧情编织师）
            return {"writer": "玄一", "confidence": 0.5}
        
        best_writer = max(scores, key=scores.get)
        confidence = scores[best_writer] / len(keywords) if keywords else 0.5
        
        return {"writer": best_writer, "confidence": min(confidence, 0.95)}
    
    def _check_poetry_requirement(self, scene_type: str, keywords: List[str]) -> bool:
        """判断场景是否需要诗词"""
        poetry_triggers = ["死亡", "告别", "战斗结束", "怀古", "意境", "悲壮"]
        return any(trigger in scene_type or trigger in keywords 
                  for trigger in poetry_triggers)
    
    def auto_sync_to_mapping_file(self, mapping: dict):
        """自动同步到 scene_writer_mapping.json"""
        mapping_file = Path("config/scene_writer_mapping.json")
        
        if mapping_file.exists():
            mappings = json.loads(mapping_file.read_text(encoding="utf-8"))
        else:
            mappings = {}
        
        # 添加新映射
        mappings[mapping["scene_type"]] = {
            "writer": mapping["writer"],
            "phase": mapping["phase"],
            "poetry_required": mapping["poetry_required"],
            "description": mapping["description"],
            "auto_generated": True
        }
        
        mapping_file.write_text(json.dumps(mappings, ensure_ascii=False, indent=2))
        
    def validate_and_confirm(self, mapping: dict) -> bool:
        """
        验证并确认映射（供人工审批）
        
        返回：
        - True: 自动置信度 >= 0.8，可自动应用
        - False: 需要人工确认
        """
        return mapping.get("confidence", 0) >= 0.8
```

#### 14.3.1 问题描述

```python
class DimensionDiscoverer:
    """维度自动发现器"""
    
    EXISTING_DIMENSIONS = [
        "case", "technique", "dialogue", "power_cost", "emotion_arc",
        "vocabulary", "character_relation", "chapter_structure", 
        "author_style", "foreshadow_pair", "worldview_element"
    ]
    
    # 维度特征模式
    DIMENSION_PATTERNS = {
        "spatial_description": {
            "keywords": ["空间", "距离", "方位", "地形", "建筑布局"],
            "min_frequency": 100,
            "description": "空间描写维度 - 描述场景空间布局和位置关系"
        },
        "timeline_tracking": {
            "keywords": ["时间", "日期", "时刻", "岁月", "光阴"],
            "min_frequency": 200,
            "description": "时间线追踪维度 - 记录故事时间节点和事件顺序"
        },
        "economic_system": {
            "keywords": ["灵石", "金币", "交易", "价格", "财富"],
            "min_frequency": 150,
            "description": "经济系统维度 - 描述交易、货币、财富流动"
        },
        "social_hierarchy": {
            "keywords": ["等级", "阶层", "地位", "身份", "血统"],
            "min_frequency": 100,
            "description": "社会阶层维度 - 描述角色社会地位和等级关系"
        }
    }
    
    def discover_new_dimensions(self, novels: List[str]) -> List[dict]:
        """
        从外部小说库发现新维度
        
        Returns:
            [
                {
                    "dimension_name": "spatial_description",
                    "frequency": 523,
                    "sample_count": 1234,
                    "confidence": 0.85,
                    "extractor_template": {...}
                }
            ]
        """
        discovered = []
        
        # 1. 分析文本特征
        text_features = self._analyze_text_features(novels)
        
        # 2. 匹配已知维度模式
        for dim_name, pattern in self.DIMENSION_PATTERNS.items():
            if dim_name in self.EXISTING_DIMENSIONS:
                continue
                
            frequency = self._count_pattern_frequency(text_features, pattern["keywords"])
            
            if frequency >= pattern["min_frequency"]:
                discovered.append({
                    "dimension_name": dim_name,
                    "frequency": frequency,
                    "sample_count": self._estimate_sample_count(novels, pattern["keywords"]),
                    "confidence": min(frequency / pattern["min_frequency"], 0.95),
                    "description": pattern["description"],
                    "keywords": pattern["keywords"]
                })
        
        # 3. 聚类发现未知维度
        unknown_dimensions = self._cluster_unknown_features(text_features)
        discovered.extend(unknown_dimensions)
        
        return discovered
    
    def generate_extractor_template(self, dimension: dict) -> dict:
        """
        自动生成维度提取器模板
        
        Returns:
            {
                "class_name": "SpatialDescriptionExtractor",
                "base_class": "BaseExtractor",
                "methods": ["extract", "parse_result", "validate"],
                "config": {...}
            }
        """
        dim_name = dimension["dimension_name"]
        class_name = self._to_class_name(dim_name)
        
        template = {
            "class_name": class_name,
            "base_class": "BaseExtractor",
            "dimension_type": dim_name,
            "keywords": dimension["keywords"],
            "methods": {
                "extract": self._generate_extract_method(dimension),
                "parse_result": self._generate_parse_method(dimension),
                "validate": self._generate_validate_method(dimension)
            },
            "config": {
                "min_content_length": 100,
                "batch_size": 50,
                "target_collection": f"{dim_name}_v1"
            }
        }
        
        return template
    
    def _to_class_name(self, dimension_name: str) -> str:
        """转换维度名为类名"""
        parts = dimension_name.split("_")
        return "".join(p.capitalize() for p in parts) + "Extractor"
```

### 14.4 遗漏L3：SKILL.md自动同步

#### 14.4.1 问题描述

**当前状态**：
- `novelist-workflow SKILL.md` 包含场景类型表格
- 发现新场景类型后，该表格未更新
- AI可能不知道新场景类型的存在

**影响**：
- AI理解偏差
- 工作流文档与实际配置不同步

#### 14.4.2 解决方案：SkillFileSyncer

```python
class SkillFileSyncer:
    """Skill文件自动同步器"""
    
    SKILL_FILE = Path("~/.agents/skills/novel-workflow/SKILL.md")
    
    # 需要同步的表格位置
    TABLE_MARKERS = {
        "scene_types": "<!-- SCENE_TYPES_TABLE -->",
        "technique_dimensions": "<!-- TECHNIQUE_DIMENSIONS_TABLE -->",
        "writer_mapping": "<!-- WRITER_MAPPING_TABLE -->"
    }
    
    def sync_scene_types_table(self, scene_types: List[dict]):
        """同步场景类型表格"""
        content = self.SKILL_FILE.read_text(encoding="utf-8")
        
        # 生成表格Markdown
        table_md = self._generate_scene_types_table(scene_types)
        
        # 替换标记位置
        marker = self.TABLE_MARKERS["scene_types"]
        if marker in content:
            content = content.replace(
                marker,
                f"{marker}\n\n{table_md}\n"
            )
        else:
            # 追加到文档末尾
            content += f"\n\n{marker}\n\n{table_md}\n"
        
        self.SKILL_FILE.write_text(content, encoding="utf-8")
    
    def _generate_scene_types_table(self, scene_types: List[dict]) -> str:
        """生成场景类型表格Markdown"""
        lines = ["| 场景类型 | 负责作家 | 需要诗词 | 描述 |", 
                 "|----------|----------|----------|------|"]
        
        for scene in scene_types:
            writer = scene.get("writer_mapping", {}).get("writer", "未分配")
            poetry = "✅" if scene.get("writer_mapping", {}).get("poetry_required") else "❌"
            desc = scene.get("description", "")[:30] + "..." if len(scene.get("description", "")) > 30 else scene.get("description", "")
            
            lines.append(f"| {scene['name']} | {writer} | {poetry} | {desc} |")
        
        return "\n".join(lines)
    
    def sync_all_tables(self, config_dir: str = "config/dimensions"):
        """同步所有表格"""
        # 1. 读取配置
        scene_types = self._load_scene_types(config_dir)
        dimensions = self._load_dimensions(config_dir)
        writer_mapping = self._load_writer_mapping(config_dir)
        
        # 2. 同步
        self.sync_scene_types_table(scene_types)
        self.sync_dimensions_table(dimensions)
        self.sync_writer_mapping_table(writer_mapping)
```

### 14.5 遗漏L4：技法-案例关联追踪

#### 14.5.1 问题描述

**当前状态**：
- 技法库：986条技法
- 案例库：38万+条案例
- 两者独立存储，无关联追踪

**影响**：
- 检索技法时，无法同时获取相关案例
- 无法知道某个技法在实际创作中的效果
- 无法识别高质量技法和低效技法

#### 14.5.2 解决方案：TechniqueCaseLinker

```python
class TechniqueCaseLinker:
    """技法-案例关联器"""
    
    def __init__(self, qdrant_client):
        self.client = qdrant_client
    
    def link_cases_to_technique(self, technique_id: str, 
                                technique_keywords: List[str],
                                top_k: int = 10) -> List[str]:
        """
        为技法关联相关案例
        
        Args:
            technique_id: 技法ID
            technique_keywords: 技法关键词
            top_k: 关联案例数量
            
        Returns:
            关联的案例ID列表
        """
        # 1. 从案例库检索相关案例
        results = self.client.search(
            collection_name="case_library_v2",
            query_vector=self._get_query_vector(technique_keywords),
            limit=top_k
        )
        
        case_ids = [r.id for r in results]
        
        # 2. 更新技法的关联案例字段
        self._update_technique_cases(technique_id, case_ids)
        
        # 3. 更新案例的关联技法字段
        for case_id in case_ids:
            self._update_case_techniques(case_id, technique_id)
        
        return case_ids
    
    def get_linked_cases(self, technique_id: str) -> List[dict]:
        """获取技法关联的案例"""
        technique = self.client.retrieve(
            collection_name="writing_techniques_v2",
            ids=[technique_id]
        )[0]
        
        case_ids = technique.payload.get("linked_cases", [])
        
        if not case_ids:
            return []
        
        cases = self.client.retrieve(
            collection_name="case_library_v2",
            ids=case_ids
        )
        
        return [c.payload for c in cases]
    
    def get_linked_techniques(self, case_id: str) -> List[dict]:
        """获取案例关联的技法"""
        case = self.client.retrieve(
            collection_name="case_library_v2",
            ids=[case_id]
        )[0]
        
        technique_ids = case.payload.get("linked_techniques", [])
        
        if not technique_ids:
            return []
        
        techniques = self.client.retrieve(
            collection_name="writing_techniques_v2",
            ids=technique_ids
        )
        
        return [t.payload for t in techniques]
    
    def batch_link_all_techniques(self, batch_size: int = 100):
        """批量关联所有技法"""
        # 获取所有技法
        techniques = self.client.scroll(
            collection_name="writing_techniques_v2",
            limit=10000
        )[0]
        
        for i, technique in enumerate(techniques):
            keywords = technique.payload.get("keywords", [])
            if keywords:
                self.link_cases_to_technique(technique.id, keywords)
            
            if (i + 1) % batch_size == 0:
                print(f"已处理 {i + 1}/{len(techniques)} 条技法")
```
