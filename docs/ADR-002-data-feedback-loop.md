# ADR-002: 数据回流闭环设计

## 状态
提议

## 上下文

当前系统的数据流是单向的：外部数据 → 提炼入库 → 检索 → 注入创作 → 输出。创作产出和评估结果没有系统化地回流到知识库中，导致：

1. 评估发现的问题无法自动沉淀为新的技法约束
2. 用户对生成内容的修改无法反馈到推荐系统
3. 技法使用效果无法追踪，无法优化推荐权重

## 决策

建立三级数据回流闭环：

### 第一级：评估反馈 → 技法库更新

```
评估结果 → 问题模式提取 → 匹配现有技法 → 更新技法使用统计/约束
```

- 评估中反复出现的同类问题，自动提取为新的技法约束
- 技法使用后的评分数据，更新技法的推荐权重

### 第二级：用户修改 → 经验沉淀

```
用户修改原文 → 差异提取 → 修改模式识别 → 经验日志记录
```

- 用户对AI生成内容的修改，通过diff分析提取修改偏好
- 高频修改模式沉淀为个人化写作经验

### 第三级：技法效果 → 推荐优化

```
技法推荐 → 创作使用 → 评估评分 → 效果追踪 → 推荐权重调整
```

- 追踪每条技法被采用后的评估得分
- 低分技法降权，高分技法升权
- 设置最小样本量阈值，避免过早得出结论

## 后果

### 积极的

1. 系统具备持续自我改进能力
2. 技法推荐越来越精准
3. 常见问题逐步被自动约束

### 消极的

1. 需要设计冲突解决策略（新旧约束矛盾时）
2. 需要阈值机制防止噪声数据污染知识库
3. 增加系统复杂度，需要监控回流质量

## 缓解措施

| 风险 | 缓解 |
|------|------|
| 噪声污染 | 设置最小样本量（≥5次）和置信度阈值（≥0.7） |
| 约束冲突 | 新约束需与已有约束做兼容性检查 |
| 回流延迟 | 异步处理，不阻塞主创作流程 |

## 验证指标

- 技法推荐命中率（采用次数/推荐次数）
- 评估分数趋势（是否随时间提升）
- 用户修改率趋势（是否随时间下降）
