# 新人快速上手指南

> 本指南帮助新用户从零构建自己的小说创作系统

---

## 一、系统概述

本系统是一个基于AI的小说创作辅助工具，核心功能：

| 功能 | 说明 |
|------|------|
| 多Agent协作 | 5位专业作家 + 1位审核评估师 |
| 知识库检索 | 设定自动检索，确保一致性 |
| 技法库检索 | 按场景/维度检索创作技法 |
| 案例库检索 | 参考标杆片段 |
| 场景映射 | 场景类型自动分配作家 |
| 经验积累 | 章节经验自动沉淀 |

---

## 二、环境准备

### 2.1 系统要求

- Python 3.9+
- Docker Desktop（运行Qdrant向量数据库）
- 8GB+ 内存（推荐16GB）

### 2.2 安装依赖

```bash
# 核心依赖
pip install qdrant-client
pip install FlagEmbedding

# 可选：格式转换
pip install ebooklib  # EPUB转换
```

### 2.3 启动向量数据库

```bash
# 拉取Qdrant镜像
docker pull qdrant/qdrant

# 启动容器
docker run -d --name qdrant -p 6333:6333 -p 6334:6334 qdrant/qdrant

# 验证
curl http://localhost:6333/collections
```

## 三、快速开始（一键构建）

```bash
# 进入项目目录
cd 你的项目目录

# 一键构建所有数据
python tools/build_all.py

# 验证
python tools/build_all.py --status
```

## 四、分步构建指南

### 第1步：初始化项目结构

```bash
python tools/init_novel.py --name "我的小说" --path "."
```

**创建的目录**：
```
我的小说/
├── 正文/                    # 已发布章节
├── 章节大纲/                # 章节规划
├── 设定/                    # 世界观/人物设定
├── 创作技法/                # 技法库
├── 章节经验日志/            # 经验沉淀
├── 写作标准积累/            # 用户修改要求
├── .vectorstore/            # 向量数据库
├── .case-library/           # 案例库
├── core/                    # 核心模块
├── modules/                 # 功能模块
├── tools/                   # 工具脚本
└── config.example.json      # 配置模板
```

---

### 第2步：构建技法库

技法库存储创作技法，按维度组织。

```bash
# 初始化技法目录结构
python tools/technique_builder.py --init

# 导入外部技法资源
python tools/technique_builder.py --import "写作技法大全.md"

# 同步到向量库
python tools/technique_builder.py --sync
```

**支持的技法维度**：

| 维度 | 覆盖范围 |
|------|----------|
| 世界观维度 | 世界构建、力量体系、势力架构 |
| 剧情维度 | 剧情设计、伏笔悬念、反转 |
| 人物维度 | 人物塑造、成长弧光、关系 |
| 战斗冲突维度 | 战斗描写、冲突升级 |
| 氛围意境维度 | 氛围营造、场景渲染 |
| 情感维度 | 情感表达、情感转折 |
| 叙事维度 | 叙事技巧、节奏控制 |
| 对话维度 | 对话技巧、潜台词 |
| 描写维度 | 描写技巧、细节刻画 |
| 开篇维度 | 开篇设计、吸引读者 |
| 高潮维度 | 高潮设计、情绪顶点 |

---

### 第3步：构建知识库

知识库存储小说设定，支持自动构建知识图谱。

```bash
# 初始化设定目录
python tools/knowledge_builder.py --init

# 编辑设定文件后，构建知识图谱
python tools/knowledge_builder.py --build-graph

# 同步到向量库
python tools/knowledge_builder.py --sync
```

**设定文件说明**：

| 文件 | 内容 | 用途 |
|------|------|------|
| 总大纲.md | 全书剧情规划 | 剧情走向参考 |
| 人物谱.md | 人物设定、弧光、关系 | 人物描写参考 |
| 十大势力.md | 势力设定、关系网络 | 势力相关章节参考 |
| 力量体系.md | 等级划分、能力类型 | 战斗/修炼描写参考 |
| 时间线.md | 事件时间顺序 | 时间线一致性检查 |
| hook_ledger.md | 伏笔追踪表 | 伏笔埋设与回收管理 |
| payoff_tracking.md | 承诺追踪表 | 对读者承诺的兑现管理 |
| information_boundary.md | 信息边界管理 | 信息差悬念管理 |

---

### 第4步：构建场景映射

场景映射决定每种场景类型由哪位作家负责。

```bash
# 初始化默认映射
python tools/scene_mapping_builder.py --init

# 查看当前映射
python tools/scene_mapping_builder.py --show

# 修改映射
python tools/scene_mapping_builder.py --set "开篇场景" "novelist-yunxi"
```

**默认场景映射**：

| 场景类型 | 负责作家 | 专长 |
|----------|----------|------|
| 开篇/结尾 | 云溪 | 意境营造 |
| 人物出场/情感 | 墨言 | 人物刻画 |
| 战斗/修炼 | 剑尘 | 战斗设计 |
| 悬念/转折/高潮 | 玄一 | 剧情编织 |
| 世界观展开 | 苍澜 | 世界观架构 |

**作家列表**：

| 作家 | Skill | 专长 |
|------|-------|------|
| 苍澜 | novelist-canglan | 世界观架构师 |
| 玄一 | novelist-xuanyi | 剧情编织师 |
| 墨言 | novelist-moyan | 人物刻画师 |
| 剑尘 | novelist-jianchen | 战斗设计师 |
| 云溪 | novelist-yunxi | 意境营造师 |

### 第5步：构建案例库（可选）

案例库存储从优秀小说提取的标杆片段。

```bash
# 初始化案例库
python tools/case_builder.py --init

# 扫描小说资源目录
python tools/case_builder.py --scan "E:/小说资源"

# 转换格式
python tools/case_builder.py --convert

# 提取案例
python tools/case_builder.py --extract --limit 5000

# 同步到向量库
python tools/case_builder.py --sync
```

---

### 第6步：初始化向量库

```bash
# 初始化所有collections
python tools/data_builder.py --init

# 查看状态
python tools/data_builder.py --status
```
