## AI助手核心能力：增删平衡

> **用户要求**：AI助手必须具备对小说的"增加"和"裁剪"能力，能够根据需要灵活调整内容，而非只会增加不会删减。

### 增删能力定义

| 能力 | 定义 | 应用场景 |
|------|------|----------|
| **增加能力** | 在情感薄弱处、主角感受不足处、氛围渲染不够处，增加必要内容 | 血牙临界点、窒息感、对父亲反应 |
| **裁剪能力** | 在内容冗余处、旁观视角过多处、写实细节过多处，删除冗余内容 | 战斗场景精简、歌词精简、解剖细节精简 |

### 增删平衡原则

#### 1. 增加原则
- **增加主角感受**：所有增加内容优先服务于主角的主观感受
- **增加情感张力**：增加"临界点"、"差点失控"等张力描写
- **增加写意表达**：用模糊意象、象征手法代替直白描写

#### 2. 裁剪原则
- **裁剪旁观视角**：删除主角不在场的详细描写
- **裁剪写实细节**：删除具体数字、解剖学细节
- **裁剪稀释内容**：删除分散主角情感的配角戏份

#### 3. 平衡原则
- **增删对等**：增加的内容应与删除的内容情感价值对等
- **总量控制**：修改后总篇幅应减少或持平，不应大幅增加
- **聚焦原则**：每次修改只解决一个核心问题，不做全面重写

### 增删操作标准

#### 判断何时增加
| 情况 | 判断标准 | 操作 |
|------|----------|------|
| 主角感受缺失 | 主角仅作为"旁观者"出现 | 增加主角的主观感受 |
| 情感无出口 | 全程压抑无临界点 | 增加"差点失控"的挣扎 |
| 氛围不足 | 写实过多导致麻木 | 增加写意化氛围渲染 |

#### 判断何时裁剪
| 情况 | 判断标准 | 操作 |
|------|----------|------|
| 配角喧宾夺主 | 配角戏份超过主角3倍 | 裁剪配角详细描写 |
| 写实过度 | 解剖学细节、具体数字过多 | 裁剪写实细节，用写意替代 |
| 内容稀释 | 集体描写分散个体情感 | 裁剪集体描写，聚焦主角 |

### 增删量化标准

| 修改类型 | 量化标准 | 示例 |
|----------|----------|------|
| **单处裁剪** | 删除5-20行，替换为1-3行 | 战斗场景从15行压缩到3行 |
| **单处增加** | 增加3-7行，不超过删除量的50% | 增加5行临界点描写 |
| **整体平衡** | 净删除行数应≥0 | 删除48行，增加14行，净减34行 |

### 增删记录格式

每轮修改需记录：

```markdown

#### 增加
| 位置 | 内容 | 行数 | 目的 |
|------|------|------|------|
| 第X行后 | [具体内容] | +N行 | [增强什么] |

#### 裁剪
| 位置 | 内容 | 行数 | 原因 |
|------|------|------|------|
| 第X-Y行 | [删除内容] | -N行 | [冗余原因] |

#### 统计
- 删除：N行
- 增加：M行
- 净变化：N-M行
```

## 标准分层使用方案

> **核心原则**：不同性质的标准，使用环节不同。硬性禁止项前置约束，软性优化项后置评估。

### 标准分层定义

| 层级 | 类型 | 定义 | 使用环节 | 处理方式 |
|------|------|------|----------|----------|
| **L1 硬性禁止** | 表达/格式错误 | 违反即为"错误" | **Generator阶段** | 必须遵守，违反则重写 |
| **L2 结构约束** | 比例/结构要求 | 可量化指标 | **Generator阶段** | 必须遵守，不达标则调整 |
| **L3 技法要求** | 写作技法建议 | 提升质量的手段 | **Generator阶段** | 强烈建议，尽力遵守 |
| **L4 质量评估** | 情感/共鸣指标 | 艺术效果评判 | **Evaluator阶段** | 评分，低于阈值建议优化 |

### L1 硬性禁止项（Generator必须遵守）

#### 表达禁止
```
❌ AI式表达：
   - "眼中闪过一丝XXX"
   - "心中涌起一股XXX"
   - "嘴角勾起一抹XXX"
   - "不禁XXX"
   
❌ 古龙式极简：
   - 词字独立成段（"痒。""疼。"）
   - 短句堆砌
   
❌ Markdown格式：
   - 中文正文使用 **XXX** 加粗
```

#### 风格禁止
```
❌ 精确年龄：
   - "十二岁的血牙" → "幼年的血牙"
   - "二十二岁的青年" → "正值青年"
   
❌ 具体数字（视情况）：
   - "三十七个男人" ✓（史诗厚重感需要）
   - "无数人" ✓（写意手法）
   - 根据场景灵活处理
```

### L2 结构约束项（Generator必须遵守）

#### 主角优先
```
✓ 主角视角占比≥15%
✓ 配角戏份不超过主角3倍
✓ 每个配角行为后，要有主角反应
```

#### 增删平衡
```
✓ 单处裁剪：删除5-20行，替换为1-3行
✓ 单处增加：增加3-7行，不超过删除量的50%
✓ 整体平衡：净删除行数应≥0
```

### L3 技法要求项（Generator强烈建议）

#### 写意手法
```
✓ 用模糊代替具体：
  - "母亲的身体在雨中摇晃" → "像一幅被风吹动的水墨画"
  
✓ 用象征代替细节：
  - 解剖细节 → "刀锋走过，带走了一部分母亲"
  
✓ 用氛围渲染代替逐句描写：
  - 歌词 → "血牙听不清歌词，只听见一个模糊的旋律"
```

#### 情感张力
```
✓ 设计"临界点"：
  - 压抑→临界点→强行压制→继续压抑
  
✓ 生理反应代替心理：
  - 不写"他很痛苦"
  - 写"他的呼吸在变慢，心跳在变慢"
```

### L4 质量评估项（Evaluator评分标准）

#### 评分维度
```
情感深度（1-10）：
  - 临界张力设计是否到位
  - 情感爆发点是否充分
  
共鸣程度（1-10）：
  - 主角痛苦是否传递给读者
  - 读者代入感强度
  
氛围渲染（1-10）：
  - 写意手法运用程度
  - 意象象征的准确性
  
节奏把控（1-10）：
  - 压抑与爆发的节奏
  - 增删修改的平衡
```

#### 阈值标准
```
✓ 通过：综合评分≥7.0
⚠️ 建议优化：5.0-7.0
❌ 需重写：<5.0
```

### 工作流程图

```
Generator阶段
    ↓
输入：大纲 + 设定 + L1/L2/L3标准
    ↓
创作：按硬性标准写作
    ↓
输出：初稿
    ↓
Evaluator阶段
    ↓
检测：L1/L2硬性项（grep检测）
    ↓ 不通过 → 返回Generator修改
评估：L4质量项（评分）
    ↓ 低于阈值 → 输出修改建议
输出：定稿
```

### 用户核心反馈

1. **"整篇总体让人在现有剧情下不够共鸣，虐不出眼泪"**
   - 不是单个情节的问题，是整篇的情感冲击力不足
   - 血牙的痛苦没有充分传递给读者

2. **"不要过多描写NPC，过多描写会让人感觉凑字数"**
   - 配角戏份稀释主角情感
   - 应聚焦主角，减少旁观视角

3. **"不要过多写具体数字，这是小说，小说讲究的是写意"**
   - 具体数字、解剖学细节过多
   - 应该用写意手法，不是写实

4. **"要着重写主角的虐"**
   - 血牙是痛苦的载体
   - 所有人物的牺牲最终都要落在血牙的"感受"上

### 诊断分析

#### 问题1：血牙的痛苦没有爆发点
- **表现**：全程克制状态，无"差点失控"的临界点
- **原因**：情感被"冰冷的理智"压制，读者感受不到张力

#### 问题2：配角稀释主角情感
- **表现**：配角行为描写占47-63行，血牙感受仅0-3行
- **原因**：读者看到的是其他人的牺牲，而不是血牙的痛苦

#### 问题3：写实过度导致麻木
- **表现**：解剖学细节、过多"看见"句式
- **原因**：写实反而削弱情感冲击力

## 精炼后的写作标准（分发给作家）

### 给所有作家的通用标准

#### 1. 主角优先原则
- 所有人物的行为/牺牲，最终都要落在主角的"感受"上
- 配角戏份不超过主角感受戏份的3倍
- 每个配角的行为后，必须有主角的反应

#### 2. 写意而非写实
- 不写具体数字（用"无数"、"很多"代替）
- 不写解剖学细节（用"消失"、"拆开"代替）
- 不写"看见XXX"句式（用感受代替视觉）
- 用模糊意象传达情感，不是细节描写

#### 3. 情感爆发点设计
- 每个压抑场景必须有"临界点"
- "想救救不了"、"想喊喊不出"的张力
- 主角的挣扎比结果更重要

---

### 给墨言（人物刻画师）的专属标准

#### 情感共鸣要点
1. **生理反应代替心理描写**
   - 不写"他很痛苦"
   - 写"他的呼吸在变慢，心跳在变慢，整个人像被什么东西扼住"

2. **临界张力设计**
   - 压抑→临界点→强行压制→继续压抑
   - 例：手指触及洞口→血脉催促冲出去→本能强行按回→没有动

3. **感受比观察重要**
   - 不写"他看见母亲的脊骨"
   - 写"他看着母亲这个'人'，在刀锋下消失"

---

### 给云溪（意境营造师）的专属标准

#### 写意技法要点
1. **用模糊代替具体**
   - 母亲的身体在雨中摇晃 → "像一幅被风吹动的水墨画"
   - 歌词逐句 → "歌声在雨中飘散，像祖灵的叹息"

2. **用象征代替细节**
   - 解剖学细节 → "刀锋走过，带走了一部分母亲"
   - 具体数字 → "无数生命，在那一夜熄灭"

3. **用氛围渲染代替逐句描写**
   - 歌词 → "血牙听不清歌词，只听见一个模糊的旋律"
   - 动作 → "她将绳索套在颈间，在赴一场归乡的旅程"

---

### 给剑尘（战斗设计师）的专属标准

#### 战斗场景要点
1. **不喧宾夺主**
   - 战斗场景篇幅不超过情感场景
   - 每个战斗行为后，要写主角的反应

2. **写意化战斗总结**
   - 不写每个人的详细战斗过程
   - 写"淡金色的光芒，一个接一个熄灭"

3. **战斗服务于情感**
   - 战斗不是目的，情感冲击才是
   - 战斗结果比过程更重要

---

### 给玄一（剧情编织师）的专属标准

#### 情感节奏要点
1. **血牙是痛苦的载体**
   - 所有剧情事件最终都要落在血牙身上
   - 读者要"跟着血牙走"，不是"看其他人"

2. **情感递进设计**
   - 压抑→临界点→强行压制→继续压抑→最终爆发
   - 不要全程压抑，要有张力

3. **配角服务于主角**
   - 配角的牺牲是为了增强主角的情感
   - 不是为了展示配角的英勇
