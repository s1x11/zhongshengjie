## 简介

天无主，地无归处。

千年的博弈从未停歇，岁月来去如风，执子之人终化作棋子，被棋局困在缝隙之间；缝隙里的他们，早把名字尘封。

众生皆苦，众生皆弈，众生在追问：我是谁？

无人应答的沉默里，风穿过石碑的纹路，穿过荒野的盐霜，穿过黎明前无人铭记的日落。

这不是答案。千年的追问，既无答案，也无尽头；却如同一枚未落定的棋子，静默地宣布自己的存在。

去问风，去问碑，去问那些死在黎明前的人。他们曾以为自己知道。

## 项目功能特色

### 多Agent协作创作系统

本项目采用Anthropic Harness架构，实现Generator/Evaluator分离的多Agent协作创作：

| 特性 | 说明 |
|------|------|
| 作家专长分工 | 5位作家各司其职：苍澜(世界观)、玄一(剧情)、墨言(人物)、剑尘(战斗)、云溪(意境) |
| 独立审核评估 | Evaluator独立评估，避免自我审查盲区 |
| 迭代反馈机制 | 自动迭代修改，最多3轮优化 |
| 经验积累系统 | 章节经验自动沉淀，后续章节复用 |

### 标准分层使用机制

用户修改要求自动分层处理：

| 层级 | 类型 | 使用环节 | 示例 |
|------|------|---------|------|
| L1 硬性禁止 | 表达/格式错误 | Generator阶段 | 禁止AI味表达 |
| L2 结构约束 | 比例/结构要求 | Generator阶段 | 主角视角>=15% |
| L3 技法要求 | 写作技法建议 | Generator阶段 | 写意手法 |
| L4 质量评估 | 情感/共鸣指标 | Evaluator阶段 | 情感深度评分 |

### 修改要求自动流转

用户反馈 -> pending状态 -> 检索时applying -> 注入作家 -> 评估后applied

### 技法向量检索

- 基于Qdrant向量数据库（ChromaDB已剔除）
- 支持维度/阶段/作家/场景过滤
- 检索优先级：数据库 -> 本地缓存 -> 文件回退

### 标杆案例库

- **256,083条**标杆案例（来自6,534本外部小说）
- 22种场景类型（打脸/高潮/开篇/战斗/对话/情感...）
- 8种题材来源（玄幻/武侠/都市/历史/科幻/校园/游戏/言情）
- 跨题材借鉴机制（历史权谋→史诗氛围，悬疑伏笔→叙事张力）

## 技术框架优势对比

### 与传统AI写作对比

| 维度 | 传统AI写作 | 本项目 |
|------|-----------|--------|
| 创作模式 | 单一模型生成 | 多Agent协作 |
| 质量保证 | 无独立审核 | Generator/Evaluator分离 |
| 经验积累 | 无记忆 | 章节经验日志 |
| 用户反馈 | 需要手动重复 | 自动记录分发 |
| 技法应用 | 依赖prompt | 向量数据库检索 |
| 迭代优化 | 手动多轮 | 自动迭代最多3轮 |

### 与其他小说AI框架对比

| 框架 | 架构 | 多Agent | 经验积累 | 用户反馈闭环 |
|------|------|---------|---------|-------------|
| 本项目 | Anthropic Harness | 5作家+1审核 | 章节日志 | 自动流转 |
| NovelAI | 单模型 | 无 | 无 | 无 |
| AI Dungeon | 单模型 | 无 | 无 | 无 |
| Sudowrite | 单模型+模板 | 部分 | 无 | 无 |
| InkOS | 多维度检测 | 检测维度 | 状态追踪 | 无 |

### 核心架构优势

1. Generator/Evaluator分离：避免自我审查盲区，提供可操作的修改建议
2. 固定3人并行前置：苍澜(世界观)+玄一(剧情)+墨言(人物)并行生成，流程统一
3. 用户修改要求闭环：pending->applying->applied状态流转
4. 经验积累机制：经验自动沉淀，后续章节自动检索

## 技术栈

Agent系统：Claude + Skill系统
向量数据库：Qdrant Docker（统一数据源 localhost:6333）
嵌入模型：BGE-M3（1024维 Dense + Sparse 混合检索）
状态管理：JSON日志
检测引擎：Grep + AST-grep
工作流：novelist-workflow

### 数据库架构

| Collection | 数据量 | 用途 |
|------------|--------|------|

| writing_techniques_v2 | 986条 | 创作技法检索 |
| case_library_v2 | 256,083条 | 标杆案例检索 |
