# 信息边界记录（Information Boundary）

> 记录《众生界》各角色知道的信息，防止信息越界（角色知道不该知道的事）。

---

## 使用说明

### 信息边界原则

| 原则 | 说明 |
|------|------|
| **信息隔离** | 角色只能知道其亲自接触/被告知的信息 |
| **传递追踪** | 信息从A→B必须有明确的传递过程 |
| **秘密保护** | 角色的秘密在被揭示前，其他角色不应知晓 |
| **预测限制** | 角色不能准确预测未来（除非有合理依据） |

### 信息类型

| 类型 | 说明 | 传递方式 |
|------|------|---------|
| **直接目击** | 角色亲眼所见 | 无需传递 |
| **直接听闻** | 角色亲耳所闻 | 无需传递 |
| **被告知** | 其他角色告知 | 需记录传递链 |
| **推断得出** | 基于已知信息推断 | 需记录推断依据 |
| **秘密** | 仅特定角色知道 | 需记录秘密持有者 |

## 信息越界检测清单

每章创作后检查：

- [x] 角色是否"记起"从未见过的事？ → 无，血牙所见均有来源
- [x] 角色是否引用了其他角色才知道的信息？ → 无
- [x] 角色是否对未知信息做出准确判断？ → 无，血牙对"血脉异常体"不懂含义
- [x] 秘密是否在被揭示前被其他角色知晓？ → 无，AI真相角色不知
- [x] 信息传递是否有明确的传递过程？ → 有，任务记录有明确捡获过程

## 常见信息越界问题

| 问题类型 | 示例 | 修正方法 |
|---------|------|---------|
| **记忆植入** | 血牙"记起"了从未见过的事 | 删除或改为被告知 |
| **信息穿透** | 角色A知道角色B的秘密（无传递过程） | 添加信息传递场景 |
| **预知未来** | 角色准确预测未来（无依据） | 改为猜测或直觉 |
| **秘密泄露** | 秘密被不该知道的角色知晓 | 检查揭示计划 |
