# 救猫咪电影剧本创作技法

> 来源：布莱克·斯奈德《救猫咪：电影编剧指南》(2005)
> 整理时间：2026-03-31

---

## 一、理论来源

《救猫咪》是好莱坞编剧布莱克·斯奈德的经典著作，核心命题：

> **"救猫咪"场景让观众立刻喜欢主角**

这一方法论系统化了商业电影剧本创作，提出15个节拍表结构，可直接转化为小说情节设计、开篇设计、角色塑造等创作技法。

---

## 二、救猫咪核心概念

### 2.1 什么是"救猫咪"

**定义**：在故事早期，主角做一些让人喜欢的事情，建立观众与角色的情感连接。

**原理**：
```
观众喜欢主角 → 愿意跟随主角经历冒险 → 产生情感投入 → 故事成功

反面案例：
× 开场主角做令人讨厌的事
× 观众对主角无好感
× 观众不愿跟随冒险
× 故事失败
```

**小说转化**：
| 应用场景 | 设计方法 |
|----------|----------|
| **开篇设计** | 让主角早期做一件让人喜欢的事 |
| **角色塑造** | 展示主角的善良、勇气或可爱之处 |
| **反派设计** | 展示反派的"人性瞬间"增加复杂度 |

### 2.2 一句话故事

**公式**：
```
一句话故事 = 反讽 + 钩子 + 画面感

示例：
"一个警察来洛杉矶探望分居的妻子，
结果她工作的办公楼被恐怖分子接管。"——《虎胆龙威》

"一个商人爱上了他雇来扮演他妻子的妓女。"——《风月俏佳人》

"一个暗恋上司的职员，发现上司正是自己的网游搭档。"——《微微一笑很倾城》
```

**小说转化**：
- 用于故事概要设计
- 用于故事概念验证
- 用于开篇钩子设计

## 三、15节拍表详解

### 3.1 节拍表总览

| 节拍 | 页码位置 | 时长占比 | 核心功能 |
|------|----------|----------|----------|
| 1. 开场画面 | 1页 | 1% | 建立基调、展示起点 |
| 2. 主题呈现 | 5页 | 4% | 暗示故事主题 |
| 3. 铺垫 | 1-10页 | 10% | 建立主角普通世界 |
| 4. 催化剂 | 12页 | 1% | 打破平衡的事件 |
| 5. 争辩 | 12-25页 | 13% | 主角犹豫是否接受 |
| 6. 第二幕开启 | 25页 | 1% | 主角接受挑战 |
| 7. B故事 | 30页 | 5% | 子情节启动 |
| 8. 游戏阶段 | 30-55页 | 25% | 核心冲突展开 |
| 9. 中点 | 55页 | 1% | 虚假胜利/失败 |
| 10. 坏蛋逼近 | 55-75页 | 20% | 外部压力增加 |
| 11. 一无所有 | 75页 | 1% | 最大失败 |
| 12. 灵魂黑夜 | 75-85页 | 10% | 主角绝望反思 |
| 13. 第二幕结束 | 85页 | 1% | 找到新方向 |
| 14. 收尾 | 85-110页 | 25% | 决战与解决 |
| 15. 终场画面 | 110页 | 1% | 展示变化完成 |

### 3.2 第一幕节拍详解

#### 节拍1：开场画面（Opening Image）

**定义**：影片第一幕，建立基调、情绪、风格，展示主角起点。

**设计要点**：
```
开场画面功能：
├── 确立故事基调（喜剧、悲剧、悬疑）
├── 介绍主要人物
├── 展示主角的前史/初始状态
└── 把观众"绑定"在座位上

关键原则：
- 开场与终场应该互相对立
- 表明故事包含的情绪变化
- 演员通常只看前10页和后10页判断是否有变化
```

**小说转化**：
```
开篇设计：
- 第一段/第一章建立基调
- 展示主角初始状态
- 设置终场的对比伏笔

案例应用：
《哈利波特》开篇：德思礼一家普通乏味的生活
vs 终场：魔法世界的冒险
```

#### 节拍2：主题呈现（Theme Stated）

**定义**：前5分钟内，有人提出问题或陈述，暗示故事主题。

**设计要点**：
```
主题呈现公式：
某人（通常不是主角）→ 提出问题/陈述 → 暗示主题 → 主角当时没当回事 → 后续有深远影响

示例：
"担心你许的愿" ——《大话王》
"骄傲会导致挫折" —— 傲慢角色
"家庭比金钱更重要" —— 家庭主题故事
```

**小说转化**：
```
主题暗示设计：
- 配角/导师在开篇提出人生哲理
- 主角不以为然
- 故事过程证明这个主题
- 结局回应主题

案例应用：
《琅琊榜》开篇：林殊被问"你究竟是谁" —— 身份认同主题
```

#### 节拍3：铺垫（Set-up）

**定义**：介绍主角和"普通世界"，建立主角的初始状态。

**设计要点**：
```
铺垫任务：
├── 介绍主角：性格、弱点、欲望
├── 建立普通世界：日常、环境、关系
├── 展示主角的问题：需要解决什么
├── 救猫咪场景：让观众喜欢主角
└── 悬念钩子：暗示即将发生的变化

人物湖模型：
初始状态（此状态）→ 变化过程 → 最终状态（彼状态）
铺垫展示初始状态，为变化做准备
```

**小说转化**：
```
前三章设计：
- 第一章：主角日常、性格展示
- 第二章：弱点暴露、人际关系
- 第三章：救猫咪场景、悬念钩子
```

#### 节拍4：催化剂（Catalyst）

**定义**：打破平衡的事件，激励事件，推动主角开始冒险。

**设计要点**：
```
催化剂类型：
| 类型 | 定义 | 案例 |
|------|------|------|
| **外部事件** | 外界变化 | 信件、访客、灾难 |
| **内在变化** | 主角内心决定 | 觉醒、顿悟 |
| **机会** | 新可能性出现 | 邀请、机遇 |

催化剂效果：
- 打破主角的普通世界
- 提出需要解决的问题
- 推动故事真正开始
```

**小说转化**：
```
激励事件设计：
- 位置：故事10%-15%
- 功能：打破平衡、提出核心问题
- 强度：必须足够强烈

案例应用：
《哈利波特》：猫头鹰送来霍格沃茨入学信
《三体》：叶文洁收到三体世界的回复
```

#### 节拍5：争辩（Debate）

**定义**：催化剂之后，主角犹豫是否接受挑战。

**设计要点**：
```
争辩公式：
催化剂 → 主角犹豫 → 权衡利弊 → 最终决定

争辩类型：
| 类型 | 主角犹豫原因 |
|------|-------------|
| **恐惧型** | 害怕未知、害怕失败 |
| **责任型** | 被现有责任束缚 |
| **满足型** | 对现状还算满意 |
| **自我怀疑型** | 不相信自己能成功 |

争辩功能：
- 展示主角内心冲突
- 增加故事张力
- 让主角的选择更有意义
```

#### 节拍6：第二幕开启（Break into Two）

**定义**：主角接受挑战，进入新世界，故事真正展开。

**设计要点**：
```
第二幕开启标志：
- 主角做出决定
- 跨过门槛进入新世界
- 故事进入核心部分

设计原则：
- 必须是主动选择（而非被动）
- 标志着主角的改变决心
- 与第一幕形成鲜明对比
```

### 3.3 第二幕节拍详解

#### 节拍7：B故事（B Story）

**定义**：子情节启动，引入帮助主角转型的角色。

**设计要点**：
```
B故事功能：
├── 提供主线之外的休息
├── 引入关键配角（导师、爱情对象、朋友）
├── 承载主题讨论
└── 与主线交织促进

B故事角色类型：
| 类型 | 功能 | 案例 |
|------|------|------|
| **导师型** | 指导主角成长 | 甘道夫、邓布利多 |
| **爱情型** | 情感支撑 | 爱情、友情 |
| **镜像型** | 反映主角问题 | 影子角色 |
```

**小说转化**：
```
支线设计：
- 位置：故事25%-30%
- 引入重要配角
- 承载部分主题
- 与主线交织

案例应用：
《哈利波特》：赫敏和罗恩成为哈利的B故事伙伴
```

#### 节拍8：游戏阶段（Fun and Games）

**定义**：故事的核心部分，主角在新世界中探索、成长。

**设计要点**：
```
游戏阶段特征：
- 占故事50%时间
- 展现"海报承诺"：观众期待的内容
- 主角与对手的初次交锋
- 技能成长、世界观展开

设计要点：
- 变化丰富，不能单调
- 有趣、有挑战、有成长
- 逐步升级的冲突
```

**小说转化**：
```
中段设计：
- 占故事最大篇幅
- 主角成长的核心阶段
- 冲突逐步升级
- 读者期待的内容展现

案例应用：
修仙小说：主角修炼、历练、战斗的成长过程
```

#### 节拍9：中点（Midpoint）

**定义**：故事中间点，虚假胜利或虚假失败，赌注升级。

**设计要点**：
```
中点类型：
| 类型 | 特征 | 后果 |
|------|------|------|
| **虚假胜利** | 主角看似赢了 | 实际埋下隐患 |
| **虚假失败** | 主角看似输了 | 实际找到希望 |

中点功能：
- 提高赌注
- 改变故事方向
- 引入新信息
- 从被动转为主动（或相反）
```

**小说转化**：
```
故事中间点设计：
- 位置：故事50%
- 转折点：改变故事走向
- 升级：提高赌注
- 揭示：新信息改变认知

案例应用：
《哈利波特1》中点：发现魔法石的秘密
```

#### 节拍10：坏蛋逼近（Bad Guys Close In）

**定义**：外部压力增加，主角面临更大挑战。

**设计要点**：
```
坏蛋逼近阶段：
├── 反派力量增强
├── 内部矛盾爆发
├── 主角遭受挫折
└── 走向"一无所有"

压力来源：
- 外部：反派、环境、对手
- 内部：团队分裂、自我怀疑
```

#### 节拍11：一无所有（All Is Lost）

**定义**：主角遭受最大失败，跌入谷底。

**设计要点**：
```
一无所有设计：
- 失败必须足够严重
- 摧毁主角的信心
- 似乎没有希望
- 为"灵魂黑夜"做准备

常见手法：
- 死亡：导师、朋友、宠物死亡
- 失去：失去最重要的东西
- 背叛：被信任的人背叛
- 失败：最关键的任务失败
```

**小说转化**：
```
低谷设计：
- 位置：故事75%
- 程度：必须是真正的绝境
- 功能：为主角的最终成长做准备

案例应用：
《哈利波特》：邓布利多死亡
《三体》：罗辑被人类抛弃
```

#### 节拍12：灵魂黑夜（Dark Night of the Soul）

**定义**：主角绝望反思，面对自己的失败和恐惧。

**设计要点**：
```
灵魂黑夜要素：
├── 绝望：彻底失去希望
├── 反思：回顾失败原因
├── 面对：正视自己的弱点
└── 转变：找到新的方向

设计原则：
- 必须是主角主动面对
- 不是外部救援
- 内在成长的关键
```

**小说转化**：
```
内心挣扎设计：
- 情感真实，让读者共鸣
- 时间充足，不要匆忙
- 转变合理，有铺垫

案例应用：
《肖申克的救赎》：安迪的"灵魂黑夜"——但他从未放弃希望
```

#### 节拍13：第二幕结束（Break into Three）

**定义**：主角找到新方向，准备最终决战。

**设计要点**：
```
第二幕结束标志：
- 主角意识到真相
- 找到解决问题的新方法
- 决定采取最后行动
- 进入第三幕决战
```

### 3.4 第三幕节拍详解

#### 节拍14：收尾（Finale）

**定义**：决战与解决，主角使用新学到的东西解决问题。

**设计要点**：
```
收尾结构：
├── 集合：召集所有帮助
├── 闯关：突破障碍
├── 高潮：最终对决
└── 解决：问题得到解决

收尾原则：
- 主角必须亲自解决问题
- 使用故事中学到的技能/知识
- 不能靠外部救援或巧合
- 回应开篇的主题
```

**小说转化**：
```
高潮设计：
- 主角必须主动
- 回应所有伏笔
- 情感饱和
- 主题呈现

案例应用：
《哈利波特1》：哈利用爱与勇气保护魔法石
```

#### 节拍15：终场画面（Closing Image）

**定义**：展示主角的变化完成，与开场形成对比。

**设计要点**：
```
终场画面功能：
├── 展示主角变化
├── 与开场形成对比
├── 回应主题
└── 提供情感满足

设计原则：
- 开场与终场互为对比
- 明确展示变化
- 给观众满足感
```

**小说转化**：
```
结局设计：
- 展示主角成长
- 与开篇呼应
- 主题升华
- 留有余味

案例应用：
《哈利波特1》终场：霍格沃茨学生踏上回家旅程
vs 开场：德思礼一家的乏味生活
```

## 四、三幕剧结构

### 4.1 三幕剧比例

```
第一幕（建置）：25%
├── 开场画面
├── 主题呈现
├── 铺垫
├── 催化剂
├── 争辩
└── 第二幕开启

第二幕（对抗）：50%
├── B故事
├── 游戏阶段
├── 中点
├── 坏蛋逼近
├── 一无所有
├── 灵魂黑夜
└── 第二幕结束

第三幕（解决）：25%
├── 收尾
└── 终场画面
```

### 4.2 英雄之旅与救猫咪对照

| 英雄之旅阶段 | 救猫咪节拍 |
|--------------|-----------|
| 普通世界 | 铺垫 |
| 冒险召唤 | 催化剂 |
| 拒绝召唤 | 争辩 |
| 跨越门槛 | 第二幕开启 |
| 试炼、盟友、敌人 | 游戏阶段+B故事 |
| 接近最深洞穴 | 坏蛋逼近 |
| 苦难折磨 | 一无所有 |
| 灵魂黑夜 | 灵魂黑夜 |
| 回报 | 第二幕结束 |
| 返回之路 | - |
| 复活 | 收尾 |
| 带着灵药返回 | 终场画面 |

## 五、小说创作应用

### 5.1 开篇设计框架

```
前三章（对应第一幕前半）：

第一章（开场画面+主题呈现）：
├── 建立基调
├── 介绍主角
├── 暗示主题
└── 钩子设置

第二章（铺垫）：
├── 主角日常展示
├── 弱点暴露
├── 人际关系
└── 救猫咪场景

第三章（催化剂前奏）：
├── 悬念加深
├── 变化暗示
└── 为催化剂做准备
```

### 5.2 中段设计框架

```
中段（对应第二幕）：

前期（游戏阶段）：
├── 主角探索新世界
├── 技能成长
├── 初步冲突
└── B故事启动

中期（中点）：
├── 关键转折
├── 虚假胜利/失败
├── 赌注升级
└── 新信息揭示

后期（坏蛋逼近→一无所有）：
├── 挫折累积
├── 内部矛盾
├── 最大失败
└── 灵魂黑夜
```

### 5.3 高潮与结局设计

```
高潮（收尾）：
├── 主角主动出击
├── 使用学到的技能
├── 最终对决
├── 问题解决
└── 主题呈现

结局（终场画面）：
├── 展示变化
├── 与开篇呼应
├── 情感满足
└── 留有余味
```

## 六、实战应用清单

### 6.1 创作前：结构规划

```
□ 一句话故事：能概括整个故事吗？
□ 开场画面：建立了什么基调？
□ 主题呈现：主题如何暗示？
□ 救猫咪场景：观众为什么喜欢主角？
□ 催化剂：什么打破平衡？
□ 中点：转折是什么？
□ 一无所有：最大失败是什么？
□ 灵魂黑夜：主角如何面对？
□ 终场画面：如何与开场对比？
```

### 6.2 创作中：节拍检查

```
□ 第一幕结束：主角是否跨过门槛？
□ B故事：是否引入关键配角？
□ 中点：赌注是否升级？
□ 一无所有：失败是否足够严重？
□ 灵魂黑夜：主角是否有内在成长？
□ 收尾：主角是否亲自解决问题？
□ 终场：是否与开场形成对比？
```

### 6.3 修订时：结构审查

| 检查项 | 标准 |
|--------|------|
| 节奏问题 | 各节拍是否在正确位置 |
