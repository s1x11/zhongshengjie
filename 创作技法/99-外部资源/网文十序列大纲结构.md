# 网文十序列大纲结构

十序列大纲是一种结构化的网文创作框架，将故事拆解为十个关键序列，确保情节推进自然、爽点分布合理。

---

## 一、核心理念

### 爽感公式

```
爽 = 压力的积累 + 冲突的深化 + 释放的满足
```

### 三要素模型

| 要素 | 说明 |
|------|------|
| **压力** | 由外部环境或内部心理带来的矛盾与困境（反派压制、规则不公、主角弱势），是情绪积累的核心来源 |
| **冲突** | 压力与释放之间的桥梁，通过主角与反派、环境或内心的对抗深化张力、推动情节递进 |
| **释放** | 压力的化解和冲突的终结，通过小爽或大爽实现情绪出口 |

### 爽点类型

| 类型 | 公式 | 说明 |
|------|------|------|
| **小爽** | 小爽 = 局部冲突 + 短期胜利 | 主角行为带来的即时胜利或短期反转 |
| **大爽** | 大爽 = 长期铺垫 + 压力爆发 + 终极胜利 | 通过长时间情绪积累，在关键节点集中爆发 |

### 压力递进公式

```
压力递进 = 初级压力（开场） + 中级压力（对抗/转折） + 高级压力（危机）
```

### 情绪曲线公式

```
情绪波动 = 压力递增 + 小爽缓冲 + 大爽释放
```

## 二、八要素提炼

在构建十序列之前，先完成八要素提炼，确保与十序列逻辑一致：

| 要素 | 要求 | 说明 |
|------|------|------|
| **人物** | 不少于100字 | 设定主角（个性、动机、背景、能力、性格变化）与配角（作用、性格） |
| **欲望** | 不少于100字 | 确定主角符合背景的欲望（如财富、权力、爱情） |
| **动作** | 不少于100字 | 设计主角为实现欲望的行动 |
| **阻碍/冲突** | 不少于100字 | 包含人物、环境、心理等因素以增强紧张感 |
| **结果** | 不少于100字 | 确定影响故事走向的结局或转折 |
| **主题** | 不少于100字 | 通过角色展现核心思想 |
| **价值观** | 不少于100字 | 借主角行为揭示价值 |
| **结局** | 不少于100字 | 呈现主角应对最终冲突并体现成长变化 |

## 三、十序列大纲

### 序列一：开场

**核心任务**：引入主角、背景与初始压力，快速吸引读者，嵌入第一小爽。

**要点**：
- 建立主角初始状态（弱势/困境）
- 展示世界观核心设定
- 设置初始压力源
- 埋下第一个小爽（主角的隐藏优势/特殊之处）

**字数要求**：不少于100字

---

### 序列二：建置

**核心任务**：展示主角目标、欲望与阻碍来源，缓和开场压力并预热小爽。

**要点**：
- 明确主角的核心欲望
- 引入主要阻碍/反派势力
- 建立主角与关键人物的关系
- 预热爽点（暗示主角潜力）

**字数要求**：不少于100字

---

### 序列三：激励事件

**核心任务**：推动主角进入主线，通过初步冲突加深张力并设置局部反击的小爽。

**要点**：
- 发生改变主角命运的事件
- 迫使主角做出关键决定
- 初步展示主角能力/金手指
- 设置第一个小高潮

**字数要求**：不少于100字

---

### 序列四：对抗

**核心任务**：主角与敌人展开多层次冲突，压力与小爽交替，推动情节递进。

**要点**：
- 多次冲突升级
- 主角尝试解决但遇到新阻碍
- 压力积累与小爽释放交替
- 深化人物关系

**字数要求**：不少于100字

---

### 序列五：转折

**核心任务**：主角在失败中调整策略，张力增强，同时通过小爽保留希望。

**要点**：
- 主角遭遇挫折或失败
- 发现新的解决路径
- 策略调整/获得新资源
- 情绪低谷后的小爽反弹

**字数要求**：不少于100字

---

### 序列六：危机

**核心任务**：主角面临最大困境，情绪张力达到顶点，为大爽埋伏笔。

**要点**：
- 危机全面爆发
- 主角陷入最困难境地
- 似乎无路可走
- 埋下翻盘伏笔

**字数要求**：不少于100字

---

### 序列七：高潮

**核心任务**：主角逆袭完成关键胜利，大爽全面爆发。

**要点**：
- 主角能力/资源全面展现
- 关键对决
- 压力集中释放
- 大爽爆发点

**字数要求**：不少于100字

---

### 序列八：解决

**核心任务**：处理高潮后的后续影响，展现主角成长，并通过小爽延续胜利感。

**要点**：
- 处理高潮遗留问题
- 展示主角成长变化
- 小爽延续胜利感
- 为后续埋下新伏笔

**字数要求**：不少于100字

---

### 序列九：结局

**核心任务**：展示主角最终状态或阶段性目标的达成，通过大爽的升华释放余韵。

**要点**：
- 达成阶段性目标
- 主角状态对比（前后变化）
- 大爽余韵
- 情感升华

**字数要求**：不少于100字

---

### 序列十：回顾

**核心任务**：总结主题或埋伏笔，为后续情节制造期待。

**要点**：
- 点明/深化主题
- 埋下新悬念
- 为续集/下一卷铺垫
- 读者期待感管理

**字数要求**：不少于100字

---

## 四、故事节奏公式

```
情节 = 10序列的逻辑框架 + 小爽的嵌套 + 大爽的爆发
```

### 爽点分布建议

| 序列 | 爽点类型 | 建议 |
|------|----------|------|
| 开场 | 小爽 | 嵌入第一小爽，吸引读者 |
| 建置 | 预热爽 | 暗示主角潜力 |
| 激励事件 | 小爽 | 局部反击 |
| 对抗 | 小爽×n | 压力与小爽交替 |
| 转折 | 小爽 | 保留希望 |
| 危机 | 无 | 压力积累到顶点 |
| 高潮 | **大爽** | 全面爆发 |
| 解决 | 小爽 | 延续胜利感 |
| 结局 | 大爽升华 | 余韵释放 |
| 回顾 | 悬念钩子 | 制造期待 |
