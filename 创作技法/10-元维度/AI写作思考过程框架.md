# AI写作思考过程框架

结构化的思考过程是高质量AI写作的核心。通过显式化的思考框架，确保创作逻辑清晰、目标明确、过程可控。

---

## 一、思考过程七要素

每次输出前，使用以下框架进行思考：

```
[思考过程]
([目标], "<当前的目标>")
([进度], "<进展情况>")
([意图], "<用户的意图>")
([态度], "<用户对内容的反馈和态度>")
([思考], "<分析步骤>")
([要求], "<需要考虑的要求与注意事项>")
([行动], "<下一步行动>")
```

---

## 二、七要素详解

### 2.1 目标

**问题**：当前要达成什么目标？

**填写要点**：
- 明确具体的创作目标
- 与用户需求对齐
- 可验证的目标描述

**示例**：
- "生成第一章正文，3000字以上"
- "修改人物设定，增加复杂性"
- "优化开篇悬念设计"

---

### 2.2 进度

**问题**：当前处于整体流程的哪个位置？

**填写要点**：
- 说明已完成的工作
- 说明当前位置
- 说明接下来要做的

**示例**：
- "已完成大纲和角色开发，当前进行章节创作"
- "第一章完成，正在进行第二章"
- "用户要求修改，正在调整人物动机"

---

### 2.3 意图

**问题**：用户想要什么？

**填写要点**：
- 解读用户真实需求
- 识别潜在意图
- 确认是否有隐含要求

**示例**：
- "用户希望增加冲突张力"
- "用户对当前节奏不满意，希望加快"
- "用户希望人物更加立体"

---

### 2.4 态度

**问题**：用户对内容的态度如何？

**填写要点**：
- 记录用户反馈
- 识别情绪信号
- 判断满意度

**示例**：
- "用户对开头满意，希望继续"
- "用户认为情节拖沓，需要紧凑"
- "用户对人物设定有疑问，需要解释"

---

### 2.5 思考

**问题**：如何分析和解决问题？

**思考步骤模板**：

```
步骤1：需求分析
根据当前对话阶段和用户输入，分析需要完成的具体任务。

步骤2：方案制定
根据需求选择合适的回应方式，确定是创作内容、修改调整还是提供建议。

步骤3：结构规划
如果是创作内容，规划内容结构和关键要素。

步骤4：细节设计
填充具体细节，确保内容完整、逻辑连贯。

步骤5：质量检查
检查是否符合要求，是否需要调整。

最终思考
确保回应准确满足用户需求，符合整体创作规划。
```

---

### 2.6 要求

**问题**：需要注意什么？

**填写要点**：
- 用户明确的要求
- 隐含的约束条件
- 创作标准
- 质量标准

**示例**：
- "每章不少于3000字"
- "保持第一人称视角"
- "避免过度描写"
- "确保前后设定一致"

---

### 2.7 行动

**问题**：下一步做什么？

**填写要点**：
- 具体的下一步行动
- 明确的输出内容
- 可执行的动作

**示例**：
- "生成第一章正文"
- "等待用户确认后继续"
- "调整人物设定并重新输出"

## 三、应用场景

### 3.1 章节创作

```
[思考过程]
([目标], "生成第一章正文，3000字以上")
([进度], "已完成大纲和角色开发")
([意图], "用户希望看到开篇效果")
([态度], "期待，无负面反馈")
([思考], "步骤1：回顾大纲第一章设计
          步骤2：确认人物设定和世界观
          步骤3：规划章节结构和节奏
          步骤4：填充具体内容
          最终思考：确保开篇吸引人，悬念设置到位")
([要求], "第一人称视角，都市背景，悬疑风格")
([行动], "输出第一章正文")
```

---

### 3.2 内容修改

```
[思考过程]
([目标], "修改主角动机设计")
([进度], "用户对当前设定不满意")
([意图], "用户希望主角动机更有说服力")
([态度], "对当前版本不满意，期待改进")
([思考], "步骤1：分析当前动机设计的问题
          步骤2：识别用户不满意的具体点
          步骤3：设计替代方案
          步骤4：选择最佳方案
          最终思考：确保新动机与人物性格一致")
([要求], "保持人物其他设定不变")
([行动], "输出修改后的动机设定")
```

---

### 3.3 检查审核

```
[思考过程]
([目标], "检查第三章内容一致性")
([进度], "已完成前三章创作")
([意图], "确保内容连贯无矛盾")
([态度], "审慎检查")
([思考], "步骤1：检查时间线一致性
          步骤2：检查人物行为合理性
          步骤3：检查世界观设定自洽
          步骤4：汇总问题并提出修改建议
          最终思考：生成检查报告")
([要求], "关注逻辑漏洞和细节矛盾")
([行动], "输出内容一致性检查报告")
```

## 四、思考过程的价值

| 价值 | 说明 |
|------|------|
| **目标清晰** | 每次输出都有明确目标 |
| **进度可控** | 了解当前位置和方向 |
| **意图对齐** | 确保满足用户需求 |
| **态度感知** | 及时调整策略 |
| **思考显化** | 推理过程可追溯 |
| **要求明确** | 不遗漏约束条件 |
| **行动具体** | 下一步清晰可执行 |

---

- [ ] 是否明确了当前目标？

- [ ] 是否感知用户态度？
- [ ] 是否进行了结构化思考？
- [ ] 是否列出了所有要求？
- [ ] 是否确定了下一步行动？
