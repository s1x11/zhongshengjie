## 十、创作检查清单

### 10.1 创作前检查

```
□ 场景目的是否明确？
□ 场景氛围是否设计？
□ 场景结构是否规划？
□ 对话功能是否分配？
□ 冲突是否有层次？
□ 信息传递是否自然？
□ 场景转换是否设计？
```

### 10.2 创作后检查

```
□ 场景是否有完整三段式结构？
□ 氛围描写是否具体而非抽象？
□ 对话是否有潜台词？
□ 冲突是否有升级过程？
□ 信息传递是否避免生硬？
□ 节奏是否匹配场景类型？
□ 场景转换是否流畅？
□ 悬念是否合理设计？
□ 禁止项是否避免？
```

## 十一、实战应用示例

```
【场景：宗门议事厅争论】

基础信息：
├── 位置：天剑宗议事厅
├── 时间：傍晚
├── 类型：冲突场景
├── 参与角色：宗主、长老A、长老B、主角
├── 目的：塑造宗门内部矛盾，传递势力危机信息

氛围设计：
├── 核心氛围：压抑、紧张
├── 环境：夕阳斜照，尘埃漂浮
├── 时间痕迹：议事厅陈旧木质，裂纹可见
├── 禁止项：避免"心中涌起一股愤怒"

场景结构：
├── 开场：宗主宣布消息，建立场景
├── 中段：长老争论，冲突升级
├── 结尾：主角表态，悬念设置

对话设计：
├── 推动情节：宗主宣布消息（30%）
├── 制造冲突：长老争论（40%）
├── 塑造角色：主角沉默后表态（20%）

冲突升级：
├── Level 1：长老A质疑消息
├── Level 2：长老B直接反驳
├── Level 3：语气升级
├── Level 4：指责对方立场
├── Level 5：宗主打断，做出决定

场景转换：
├── 与上一场景：直接转换，从主角到达开始
├── 与下一场景：悬念转换，留下主角选择悬念

信息传递：
├── 情节信息：宗门面临危机（自然嵌入争论）
├── 人物信息：各方立场（通过争论体现）
├── 世界信息：势力关系（通过争论透露）

检查清单：
├── ✅ 氛围具体描写
├── ✅ 冲突有升级
├── ✅ 信息自然传递
├── ✅ 悬念设置合理
├── ✅ 禁止项避免
```
